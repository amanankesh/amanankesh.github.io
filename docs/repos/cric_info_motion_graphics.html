Cricket Info Motion Graphics

A modular pipeline that turns raw cricket match footage into a stylized “chalkboard / motion graphics” look: player and ball emphasis, static overlay removal, edge illustration, compositing, and a final cinematic pass.

View on GitHub ← Back to home

Outputs

Each full run writes to a timestamped folder under outputs/. Depending on which steps you enable, you get:

  • Per-frame masks for players and ball (segmentation steps).
  • A cleaned source plus a static mask for logo/banner-style regions.
  • An edge-map video on a chalk-style background.
  • A highlighted composite (edges + masks + background).
  • A graded final.mp4 with film-style post (darken, warmth, glow, grain, flicker).

Pipeline modules

The orchestrator in main.py runs steps in a fixed order so masks exist before compositing:

Step Module Role
1 steps/player_segmentation.py YOLO prompts + SAM2 video propagation → player masks.
2 steps/ball_segmentation.py Ball box search + SAM2 propagation → ball masks.
3 steps/static_removal.py Bottom banner strip + temporal static logo estimate → mask PNG + cleaned MP4.
4 steps/edge_map.py Canny + skeletonization + chalk background → edge map video.
5 steps/highlighting.py Composite edge map with masks and background image.
6 steps/effects_step.py Final look: grade, glow, grain, flicker → final.mp4.

Execution order in production: static removal first → players → ball → edge map → highlighting → effects.

Stack

Python · YOLO · SAM2 · OpenCV-style image ops · configurable outputs/ layout for repeatable runs.
