Generalizable Gaussian Avatar

Research codebase for dynamic Gaussian head avatars in the GAGAvatar family: train on tracked video, run inference with a reference identity and driver frames, and optionally fine-tune or use combined training paths.

View on GitHub ← Back to home

What you can do

  • Train — Lightning Fabric loop with TensorBoard; configs split by model and data (e.g. FLAME, camera, dataset paths).
  • Inference — inference.py for reference + driver frames (forward_expression).
  • Video-style transfer — vid2vid_infer.py for video-to-video style runs.
  • Tracking stack — Face tracking and matting live in the GAGAvatar_track submodule.

Repository map (abbrev.)

  • configs/model, configs/data — model name, optimizer, schedule, dataset and FLAME/camera settings.
  • core/models/GAGAvatar — main model exports (e.g. CombinedModel.GAGAvatar).
  • core/libs/flame_model — FLAME mesh helpers; core/libs/GAGAvatar_track — tracking engines.
  • train.py, CombinedTraining.py, FineTuning.py — training entry points.
  • build_resources.sh — pulls FLAME/GAGAvatar assets (license prompts as upstream requires).

Stack

Python · PyTorch · Lightning Fabric · FLAME · 3D Gaussian rendering · TensorBoard
