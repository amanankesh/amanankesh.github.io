Sports Video Search

Production-oriented MVP for hybrid semantic + structured search over sports clips. Vertex AI powers query parsing and answer summarization; PostgreSQL applies metadata filters; FAISS retrieves by embedding similarity. FastAPI exposes /search and health endpoints suitable for containers and Google Cloud Run.

Request path (conceptual)

  1. Query understanding — Gemini reads the user query using prompt/query_parser_prompt.txt and emits JSON facets. If Vertex is off or errors, search can still run on the raw string with empty facets.
  2. Retrieval — Postgres applies structured filters when facets exist; FAISS returns top‑K by cosine-style similarity (L2-normalized inner product). Results merge and dedupe by clip id.
  3. Ranking — Weighted blend of editorial signals (action_score, visual_quality, …), vector similarity, and facet match bonuses.
  4. Answer — Gemini summarizes the top hits in neutral, sports-appropriate wording.

Startup (FastAPI lifespan)

On load, the app configures logging, pydantic settings, optional vertexai.init, and loads the FAISS index + metadata if paths are valid and consistent. Optional CREATE_TABLES_ON_STARTUP ensures ORM tables exist. CORS is applied when CORS_ORIGINS is set.

API surface

Search is implemented under both GET /search and GET /v1/search via a shared router in app/routes/search.py — check the repo for the exact request/response schema and env vars (FAISS_INDEX_PATH, GOOGLE_CLOUD_PROJECT, database URL, etc.).

Stack

Python · FastAPI · Vertex AI (Gemini + embeddings) · PostgreSQL · FAISS · Google Cloud Run
