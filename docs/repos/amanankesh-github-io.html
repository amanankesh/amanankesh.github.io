What this repo is for

GitHub Pages serves the main branch (or your configured publishing source). This tree holds the hand-written pages you are reading: no build step required, easy to version and review in PRs.

Layout

  • docs/index.html — main portfolio: about, repo cards, featured write-ups, skills.
  • docs/css/site.css — shared typography, cards, buttons, and article styles.
  • docs/repos/*.html — one page per public repository with summary + GitHub link.
  • docs/*.html — longer narrative pages (CodePilot, video agent, etc.).
  • Root index.html — minimal entry that points visitors to docs/ when needed.

Why static HTML

Fast loads, straightforward hosting on GitHub Pages, and full control over markup and accessibility. When a project grows, you can still add a separate app or docs site and link it from here.

Stack

HTML · CSS · GitHub Pages (no client-side framework in the baseline layout).
